# A Simulation Study of Proximity-Aware Load Balancing on Datacentre Networks

A custom C++ discrete-event simulator, accompanied by a Python
analysis and plotting pipeline, for evaluating proximity-aware
variants of the Power-of-*d*-choices load-balancing policy on
structured server topologies.

The simulator is the one used for the MEng project
*A Simulation Study of Proximity-Aware Load Balancing on
Datacentre Networks* (Dahmani, Durham University).

---

## Quick start

```sh
git clone <this-repo> scheduler-sim-final
cd scheduler-sim-final
make                                   # builds bin/loadbal_sim
./bin/loadbal_sim --help 2>/dev/null || true

# validate that the simulator matches the Po-d closed form
python scripts/validate_pod.py

# run the full synthetic parameter sweep
python scripts/run_sweep.py

# produce the publication figures from the sweep results
python scripts/figures_script.py
```

All generated artefacts (the binary, results, figures, parsed
traces) live in directories that are excluded from version
control; see `.gitignore`.

## Overview

The simulator places *N* homogeneous servers on a network topology
(cycle, grid, or partitioned cluster) and routes incoming jobs
according to a load-balancing policy. Two policies are compared
throughout the study:

| Policy      | Description                                           | `k`     | `L`     |
|-------------|-------------------------------------------------------|---------|---------|
| `poKL`      | Global Power-of-*d*: all probes are uniformly random  | `0`     | `d - 1` |
| `spatialKL` | Proximity-aware hybrid: local neighbours + one global | `d - 2` | `1`     |

At each arrival the origin server probes `1 + k + L = d` candidates
(including itself), and the job is forwarded to whichever
candidate currently has the shortest queue.

Three network topologies are supported:

- **cycle** &mdash; *N* nodes arranged in a ring; distance is the
  shortest arc length along the ring.
- **grid** &mdash; *N* nodes on a rectangular 2-D grid; distance is
  the Manhattan (L&sup1;) norm with no wrap-around.
- **cluster** &mdash; *N* nodes partitioned into fully connected
  clusters; same-cluster distance is one hop, different-cluster
  distance is two hops.

The simulator is a next-event discrete-event engine: at each step
it selects the earliest of (next arrival, earliest service
completion), advances the clock exactly to that event, and updates
state. There is no fixed-step discretisation.

## Requirements

- A C++17 compiler (tested with `g++`)
- `make`
- Python 3.9+ with the following packages for the analysis scripts:
  `numpy`, `pandas`, `matplotlib`

make               # compiles bin/loadbal_sim

## Running the Simulator

The binary is invoked with command-line flags. Every parameter has
a sensible default, so a minimal invocation is valid:

```sh
./bin/loadbal_sim
```

| Flag         | Default       | Description                                                          |
|--------------|---------------|----------------------------------------------------------------------|
| `--n`        | `1000`        | Number of servers                                                    |
| `--m`        | `100000`      | Total number of jobs to simulate                                     |
| `--lambda`   | `0.9`         | Per-server arrival rate (0 &lt; &lambda; &lt; 1)                     |
| `--mu`       | `1.0`         | Per-server service rate                                              |
| `--policy`   | `pot`         | One of `pot`, `poKL`, `spatialKL`                                    |
| `--topo`     | `cycle`       | One of `cycle`, `grid`, `cluster`                                    |
| `--k`        | `1`           | Number of local neighbours probed (`spatialKL`)                      |
| `--L`        | `1`           | Number of global random probes                                       |
| `--clusters` | `1`           | Number of clusters (only used when `--topo cluster`)                 |
| `--cost`     | `0.0`         | Inter-cluster communication cost multiplier                          |
| `--trace`    | *(empty)*     | Path to a `.dat` workload trace; overrides the synthetic generator   |
| `--seed`     | `123456789`   | RNG seed (for replication studies)                                   |
| `--outdir`   | `results`     | Directory to write results into                                      |
| `--tag`      | *(empty)*     | Optional suffix appended to output filenames                         |

Example: a 525-server cluster run with the spatial policy at
*d* = 3, under load &lambda; = 0.95, writing results to
`./results_demo/`.

```sh
./bin/loadbal_sim --n 525 --m 10000000 --lambda 0.95 \
                  --policy spatialKL --topo cluster --clusters 21 \
                  --k 1 --L 1 \
                  --outdir results_demo --tag demo
```

## Output Format

Each run writes two files into `--outdir`:

- `<policy>_<topo>_n<N>_lam<lambda>[_<tag>]_metrics.json` &mdash; the
  aggregate metrics from the run:

  ```json
  {
    "policy": "spatialKL",
    "graph":  "cluster",
    "n": 525,
    "m": 10000000,
    "lambda": 0.95,
    "mu": 1,
    "k": 1, "L": 1, "qmax": 100,
    "num_clusters": 21, "comm_cost": 0,
    "total_req_dist": 7191,
    "mean_Q": 1.97792,
    "mean_W": 351.402,
    "avg_req_dist": 0.726364
  }
  ```

- `<policy>_<topo>_n<N>_lam<lambda>[_<tag>]_hist.csv` &mdash; the
  post-warm-up time-weighted queue-length distribution
  P(Q = *k*):

  ```csv
  QueueLength,Probability
  0,0.342534
  1,0.120361
  ...
  ```

When `--trace` is supplied, the filename substitutes `_trace` for
`_lam<lambda>`.

### Metric definitions

| Field            | Definition                                                                                   |
|------------------|----------------------------------------------------------------------------------------------|
| `mean_Q`         | Time-weighted mean queue length E[*Q*] across all servers                                    |
| `mean_W`         | Mean response time E[*R*], derived from E[*Q*] via Little's law                              |
| `avg_req_dist`   | Mean communication cost E[*c*]: average topological distance from origin to chosen target   |
| `total_req_dist` | Sum of origin-to-target distances over all post-warm-up arrivals                             |

## Python Tooling

The `scripts/` directory contains a small analysis pipeline that
treats the simulator as a black box and reads back its JSON and
CSV outputs. A natural workflow is:

### 1. Validate the simulator &mdash; `validate_pod.py`

Runs the global Po2 configuration on all three topologies and
overlays the empirical queue-length distribution on the closed-form
curve *P*(*Q* &ge; *i*) = &lambda;<sup>(2<sup>i</sup>&minus;1)</sup>.
The three empirical curves should collapse onto the analytical one.

```sh
python scripts/validate_pod.py
```

Writes into `results_validation/` and `figures/validation_pod.png`.

### 2. Sanity-check one configuration &mdash; `quick_compare.py`

For a single `(topology, d)`, sweeps &lambda; on a small run
(*M* = 10<sup>7</sup>) and produces a two-panel plot showing E[*R*]
and E[*c*] for both policies. Fast feedback before launching the
full sweep.

```sh
python scripts/quick_compare.py --topo grid --d 4
```

### 3. Full synthetic sweep &mdash; `run_sweep.py`

Runs every
`(topology, policy, lambda, d)` configuration in parallel and
writes per-configuration metrics to
`results_topology_sweep/<topo>/`. Any subset of the three
topologies can be selected.

```sh
python scripts/run_sweep.py                  # all three topologies
python scripts/run_sweep.py --topos cycle    # cycle only
python scripts/run_sweep.py --workers 2      # throttle CPU usage
```

### 4. Publication figures &mdash; `figures_script.py`

Reads the results produced by step&nbsp;3 and generates the IEEE-style
PDF figures for the Results section of the report.

```sh
python scripts/figures_script.py --results-root results_topology_sweep
```

### 5. Trace-driven experiments &mdash; `process_traces.py` and `trace_experiment.py`

`process_traces.py` converts raw Google Borg (2019) or Alibaba
(2025) cluster trace CSVs into the `(inter_arrival_time, duration)`
`.dat` format expected by the simulator's `--trace` option.
`trace_experiment.py` then runs both policies against a given
trace and produces a comparison plot.

```sh
python scripts/process_traces.py --source alibaba raw.csv data/alibaba_test/alibaba_trace.dat
python scripts/trace_experiment.py data/alibaba_test/alibaba_trace.dat --label alibaba
```

### 6. Seed-replication study &mdash; `replication_study.py`

Re-runs a small set of sentinel configurations under several
different RNG seeds and reports the standard deviation of each
metric, verifying that the main-sweep results are stable under
stochastic variation.

```sh
python scripts/replication_study.py                  # 10 seeds
python scripts/replication_study.py --seeds 20
```

## Reproducibility

All randomness in the simulator originates from a single
`std::mt19937_64` Mersenne Twister generator, seeded at
construction. By default the seed is fixed at `123456789`, so any
two invocations with identical parameters produce bit-identical
output. The `--seed` flag exposes the seed for replication
studies; the Python script `replication_study.py` uses this to
sweep seeds systematically.
